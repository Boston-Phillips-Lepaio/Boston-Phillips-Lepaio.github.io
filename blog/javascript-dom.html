HTML would be like a structure of a gun, the makings of it like triggers and magazines. The CSS would be like the camouflage of the gun and the shape of it too, giving its appearance. JavaScript is like what the gun does when you interact with it, for example pulling on the trigger one time will shoot one bullet and holding it will shoot many.

Control flow is how your code flows from top to bottom. When washing your hands the control flow is: Apply soap, add water, rub water and soap over hands, rinse off excess soap, dry hands. Looping in the same context would be like washing your hands in a circular motion. Once this wash loop is over, move on to rinsing off excess.

The DOM (Document Object Model) is something where you can display and access the structure of a document in a tree format where everything stems from the HTML (but really window). You can use JavaScript to interact with it for example:

document.getElementById("OLDTEXT").innerHTML = "NEWTEXT"

This command is used to access an element by its ID and change it from OLDTEXT to NEWTEXT.

Object are like a set of lockers. Each locker has a name and you can access items in the locker using the names (keys) assigned to the lockers. This is different to arrays which treat it more like numbered lockers that begin from 0, which are instead accessed using that number.

A function is a specific structure of code that will perform a "function" for you, the same way, every time. The "portAFort" function in Fortnite is a reusable structure that can be deployed whenever you need, very helpful.
