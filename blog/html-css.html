Although Classes and IDs can be very similar in nature, there are a few key differences that discern what they are and their uses.

Classes are used for styling a group of similar elements all at once. For example if you had a grid with a multitude of lists, paragraphs, headings and whatnot and you wanted to change the color of all these different texts, instead of styling them inline on the html or once at a time in the style sheet, you can instead create a class that encompasses the content within the grid - allowing you to change the color of everything within said class at once.

IDs differ from classes in which they are intended to be used on SPECIFIC html elements instead of grouped. Also unlike classes, IDs must be unique. For example, if there is a specific element on a webpage like the main heading that requires a distinct color; you can assign an ID to that element ensuring the style is uniquely associated with that particular element. IDs are also majorly used in javascript which I haven't dived into yet.
